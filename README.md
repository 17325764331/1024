# 2048 (纯 HTML)

一个无需构建工具、可直接运行的 2048 小游戏。

## 运行方式

1. 直接双击 `index.html` 在浏览器打开；或
2. 使用任意静态服务器在项目根目录启动后访问页面。

## 操作方式

- 桌面端：`↑ ↓ ← →` 或 `W A S D`
- 移动端：在棋盘区域滑动
- 点击“新游戏”可重开
- 达成 2048 后可点击“继续游戏”冲击更高分

## 已实现功能

- 4x4 / 5x5 棋盘切换
- 四方向移动、合并、计分
- 合并规则：单回合单次合并
- 最高分本地持久化（按 4x4 / 5x5 分开统计）
- 胜利/失败提示与重开流程
- 桌面键盘 + 移动端滑动输入
- 最近 20 局历史记录（按 4x4 / 5x5 分开统计）
- PWA 支持（可安装到手机桌面，支持离线缓存）

## 手工验证清单

- 新开局有 2 个数字块
- 无效移动不生成新块
- `[2,2,2,2]` 一次移动结果应为 `[4,4,0,0]`（任意尺寸）
- 达成 2048 时显示提示并可继续
- 无可移动空间时显示“游戏结束”
- 刷新页面后当前尺寸的最高分与历史保留
- 切换尺寸后，显示对应尺寸的独立最高分与历史

## 已知限制

- 当前不支持 Undo（撤销一步）
- 仅本地存储，不同设备/浏览器之间不自动同步

## 发布到 GitHub Pages（可安装到手机桌面）

1. 将项目推送到 GitHub 仓库，默认分支为 `main`。
2. 仓库中已包含工作流：`.github/workflows/deploy-pages.yml`。
3. 打开 GitHub 仓库页面：`Settings` → `Pages`，确保 Source 为 `GitHub Actions`。
4. 每次 push 到 `main` 会自动发布，访问：
   - `https://<你的用户名>.github.io/<仓库名>/`
5. 手机上通过该 `https` 链接打开后可“添加到主屏幕”。

说明：

- 直接发送单个 `html` 文件（`file://` 打开）可以玩，但通常不能作为 PWA 安装。
- 安装能力依赖 `https`、`manifest.webmanifest`、`sw.js` 等文件协同工作。
