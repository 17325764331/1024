# 2048 Product Steering

## Product Vision
Build a lightweight and maintainable 2048 game project with clear feature evolution and stable user experience.

## Target Users
- Casual players: want quick and smooth play sessions.
- Developers: want easy extension and low onboarding cost.

## User Problems
- Inconsistent behavior after feature changes.
- Missing requirement traceability for game rule updates.
- Hard to verify whether changes are safe to release.

## Product Principles
- Playability first: keep controls and feedback intuitive.
- Predictability: rule behavior must be deterministic.
- Evolvability: new features should not break existing game flow.

## Core Features
1. Core 2048 gameplay loop
2. Score and move feedback
3. Stable release and rollback process

## Scope Boundaries
### In Scope
- Spec-driven planning for gameplay and UI changes.
- Requirement, design, task, and implementation logging.

### Out of Scope
- Large platform-level infrastructure changes unrelated to gameplay.
